# LEXXI PRO - PATCH TEST SYSTEM

## 🧪 PATCH TEST REQUIREMENTS

### **WHEN PATCH TESTS ARE MANDATORY**

**Legal Requirements (varies by state, but best practice):**
- ✅ First-time clients (never used salon before)
- ✅ New product line (switching from Wella to Redken)
- ✅ New chemical service type (first time doing lightener)
- ✅ Clients with known allergies or sensitivities
- ✅ Clients with previous adverse reactions
- ✅ After 12+ months since last color service
- ✅ Fashion/vivid colors (higher allergen risk)
- ✅ Bleach/lightener services on previously untreated hair

**Recommended (but not always required):**
- Clients with eczema, psoriasis, or sensitive skin
- Pregnant clients (extra caution)
- Clients on new medications
- Corrective color work with multiple products

## 🗄️ DATABASE SCHEMA

```sql
CREATE TABLE patch_tests (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  client_id UUID REFERENCES clients(id),
  stylist_id UUID REFERENCES stylists(id),
  
  -- Scheduling
  test_date TIMESTAMP NOT NULL,
  service_date TIMESTAMP NOT NULL,
  
  -- Products tested
  products JSONB NOT NULL, -- Array of {brand, product, location}
  
  -- Pre-test screening
  screening_responses JSONB, -- Answers to questionnaire
  contraindications TEXT[],
  
  -- Application
  application_photos JSONB, -- Array of photo URLs
  application_time TIMESTAMP,
  application_location TEXT[], -- ["behind_right_ear", "behind_left_ear"]
  
  -- Monitoring
  checkin_8hr TEXT, -- Client response
  checkin_24hr TEXT,
  checkin_45hr TEXT,
  
  -- Result
  status TEXT CHECK (status IN ('pending', 'passed', 'failed', 'cancelled')),
  reaction_type TEXT, -- If failed: "mild", "moderate", "severe"
  reaction_notes TEXT,
  
  -- Verification (day of service)
  verified_by UUID REFERENCES stylists(id),
  verified_at TIMESTAMP,
  verification_photos JSONB,
  
  -- Consent
  consent_pdf_url TEXT,
  client_signature_url TEXT,
  stylist_signature_url TEXT,
  
  created_at TIMESTAMP DEFAULT NOW(),
  
  INDEX idx_client (client_id),
  INDEX idx_service_date (service_date),
  INDEX idx_status (status)
);
```

## 📱 COMPONENT IMPLEMENTATION

```typescript
// src/components/PatchTestScheduler.tsx

import React, {useState} from 'react';
import {View, Text, TouchableOpacity, StyleSheet, Alert} from 'react-native';
import DateTimePicker from '@react-native-community/datetimepicker';

interface Product {
  brand: string;
  name: string;
  type: 'lightener' | 'color' | 'toner' | 'bond_builder';
}

interface PatchTestSchedulerProps {
  clientName: string;
  serviceDate: Date;
  products: Product[];
  onSchedule: (testDate: Date) => void;
}

export default function PatchTestScheduler({
  clientName,
  serviceDate,
  products,
  onSchedule,
}: PatchTestSchedulerProps) {
  const minDate = new Date();
  const maxDate = new Date(serviceDate);
  maxDate.setHours(maxDate.getHours() - 48); // 48 hours before service

  const [testDate, setTestDate] = useState(maxDate);

  const handleSchedule = () => {
    const hoursDiff = (serviceDate.getTime() - testDate.getTime()) / (1000 * 60 * 60);
    
    if (hoursDiff < 48) {
      Alert.alert('Invalid Date', 'Patch test must be at least 48 hours before service');
      return;
    }
    
    onSchedule(testDate);
  };

  return (
    <View style={styles.container}>
      <View style={styles.warning}>
        <Text style={styles.warningIcon}>⚠️</Text>
        <Text style={styles.warningText}>PATCH TEST REQUIRED</Text>
      </View>

      <View style={styles.section}>
        <Text style={styles.label}>Reason:</Text>
        <Text style={styles.value}>First-time client</Text>
        <Text style={styles.value}>New product line</Text>
      </View>

      <View style={styles.section}>
        <Text style={styles.label}>Products to Test:</Text>
        {products.map((product, i) => (
          <Text key={i} style={styles.product}>
            • {product.brand} {product.name}
          </Text>
        ))}
      </View>

      <View style={styles.section}>
        <Text style={styles.label}>Service Date:</Text>
        <Text style={styles.serviceDate}>
          {serviceDate.toLocaleDateString()} at {serviceDate.toLocaleTimeString()}
        </Text>
      </View>

      <View style={styles.section}>
        <Text style={styles.label}>Patch Test Must Be Completed By:</Text>
        <Text style={styles.deadline}>
          {maxDate.toLocaleDateString()} at {maxDate.toLocaleTimeString()}
        </Text>
      </View>

      <View style={styles.section}>
        <Text style={styles.label}>Schedule Patch Test Appointment:</Text>
        <DateTimePicker
          value={testDate}
          mode="datetime"
          minimumDate={minDate}
          maximumDate={maxDate}
          onChange={(event, date) => date && setTestDate(date)}
        />
      </View>

      <TouchableOpacity style={styles.button} onPress={handleSchedule}>
        <Text style={styles.buttonText}>SCHEDULE PATCH TEST (15 min)</Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {padding: 20, backgroundColor: '#fff'},
  warning: {flexDirection: 'row', alignItems: 'center', backgroundColor: '#FFF3CD', padding: 16, borderRadius: 8, marginBottom: 20},
  warningIcon: {fontSize: 24, marginRight: 12},
  warningText: {fontSize: 16, fontWeight: 'bold', color: '#856404'},
  section: {marginBottom: 20},
  label: {fontSize: 12, color: '#666', marginBottom: 8, fontWeight: '600'},
  value: {fontSize: 14, color: '#333', marginBottom: 4},
  product: {fontSize: 14, color: '#333', marginLeft: 8, marginBottom: 4},
  serviceDate: {fontSize: 18, fontWeight: 'bold', color: '#D4AF37'},
  deadline: {fontSize: 16, fontWeight: 'bold', color: '#dc3545'},
  button: {backgroundColor: '#D4AF37', padding: 16, borderRadius: 8, alignItems: 'center'},
  buttonText: {fontSize: 16, fontWeight: 'bold', color: '#fff'},
});
```
